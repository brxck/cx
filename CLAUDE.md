# cx

CLI for orchestrating Cmux layouts on top of Coder remote dev environments.

## Project docs

- `DESIGN.md` ‚Äî architecture, command reference, Cmux integration, SSH socket forwarding, template system, state storage
- `PROGRESS.md` ‚Äî implementation status of all commands, features, and infrastructure

Read these before making changes.

## Stack

- **Runtime**: Bun (CLI runtime + bundler ‚Äî `bun:sqlite`, `Bun.$`, `Bun.serve`, `bun build --compile`)
- **Package manager**: pnpm workspaces (only manages packages and the lockfile; runtime is still Bun)
- **Arg parsing**: Citty ‚Äî `defineCommand` / `runMain`
- **Prompts**: @clack/prompts
- **Logging**: consola
- **Colors**: picocolors
- **State**: bun:sqlite at `~/.cx/state.db`
- **Build**: `bun build --compile` for standalone binaries

## User config

- Personal configurations live at `~/.config/cx`

## Project layout

This is a pnpm monorepo. Packages live under `packages/`:

- `packages/cli/` ‚Äî the `cx` CLI binary
  - `packages/cli/src/cli.ts` ‚Äî main entry, defines root command and subcommands
  - `packages/cli/src/commands/*.ts` ‚Äî one file per subcommand, each exports a `defineCommand`
  - `packages/cli/src/lib/store.ts` ‚Äî SQLite state store (layouts, sessions), migrations via `PRAGMA user_version`
  - `packages/cli/src/lib/cmux.ts` ‚Äî Cmux CLI wrapper (workspaces, panes, surfaces, input, notifications)
  - `packages/cli/src/lib/templates.ts` ‚Äî template types, load/save, per-project discovery, cmux.json generation
  - `packages/cli/src/lib/coder.ts` ‚Äî Coder CLI wrapper (list, create, start, stop, wait, SSH, config-ssh)
  - `packages/cli/src/lib/workspace-picker.ts` ‚Äî shared interactive workspace picker
  - `packages/cli/src/api/*.ts` ‚Äî HTTP handlers exposed by `cx serve`
  - `packages/cli/src/web/embedded.ts` ‚Äî generated by `scripts/embed-web.ts`, gitignored
  - `packages/cli/bin/cli.ts` ‚Äî shebang entrypoint
  - `packages/cli/scripts/embed-web.ts` ‚Äî reads `../../web/dist`, writes `src/web/embedded.ts`
- `packages/web/` ‚Äî embedded React UI served by `cx serve`; built by Vite, output piped into the CLI
- `packages/api-types/` ‚Äî `@cx/api-types`, the shared HTTP API contract; CLI re-exports, Raycast imports
- `packages/raycast/` ‚Äî Raycast extension that drives `cx serve` over HTTP

## Scripts

Run from the repo root:

- `pnpm dev` ‚Äî run CLI in dev mode (`pnpm --filter cx dev`)
- `pnpm build` ‚Äî build web, embed assets, compile standalone binary to `packages/cli/dist/cx`
- `pnpm typecheck` ‚Äî `pnpm -r typecheck` across all packages
- `pnpm dev:raycast` ‚Äî `ray develop` in `packages/raycast`

Run `pnpm build` after finishing a CLI change to confirm it compiles.

## Terminology

- **Workspace** ‚Äî a Coder remote dev environment
- **Layout** ‚Äî a Cmux workspace (renamed to avoid collision with Coder's "workspace")
- **Template** ‚Äî config that defines a layout: Coder workspace params + Cmux split/pane/surface tree

## Conventions

- Use Bun APIs: `Bun.$` for shell, `Bun.file` for file I/O, `bun:sqlite` for SQLite
- Shell commands via `Bun.$\`cmd\`.quiet()` for captured output, `Bun.spawn()` for interactive/long-running processes
- All store functions are synchronous (bun:sqlite is sync)
- Commands follow the pattern: resolve target (arg or interactive picker) ‚Üí do work with spinners ‚Üí save state
- Cmux CLI has no JSON output for `list-workspaces`; we parse text output (see `parseWorkspaceLine` in cmux.ts)
- Templates use the same recursive split tree format as Cmux custom commands (cmux.json)
- Generated cmux.json entries are tagged with `_generator: "cx"` for safe merge/cleanup
